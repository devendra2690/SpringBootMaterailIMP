Error Codes and Responses



HTTP Status Codes

The Revspring API attempts to return appropriate HTTP status code for every request

Code Text Description
200 OK Request is Successful
400 Bad Request The request was invalid. An accompanying error message will explain further
401 Unauthorized Missing or incorrect authentication credentials.
403 Forbidden The request is understood, but it has been refused or access is not allowed with given credentials
404 Not Found The URI requested is invalid or the resource requested or document does not exists
406 Not Acceptable Returned when an invalid format is specified in the request.
500 Internal Internal Server Error Something is broken. Please contact RevspringInc Customer care


Sample Error Response

The error response is in JSON format. For instance, error might look like this :

{
"code": "1006",
"message": "Required Fields are missing",
"status": "400",
"timestamp": "Wed Jul 05 11:06:14 CDT 2017",
"errors": [{
   "code" : "ERR003"
   "message" : "accountNumber or parentNumber required"
      }]
}



Error Codes

The following table describes the high level error codes which may appear when working with the API

Code HTTP Status Code Text Description
1001 400 Data Validation Error Invalid Format
1002 403 Authorization Error Client is not authorized to perform this transaction
1003 500 System Error Web service is down or broken
1004 500 Configuration Error Client Configuration error. Please contact Client configuration team
1006 400 Required Fields are missing Required fields are missing to perform this API call
1007 404 Not Found data not found
